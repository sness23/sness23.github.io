<h3 class="title"></h3>

<p>Two cute grep commands for protein structures in .pdb files:</p>

<p><b>1) Count the number of residues in a .pdb file</b><br />
For this one, I strongly suggest using the first command with "less" first, to make sure there aren't any weirdly named waters or other co-factors.  If there are, just add them to the "grep -v" section.</p>

<p>Explanation:<br />
First, we remove the REMARK card.  Then we grep for an ATOM or HETATOM (hetero-atom) card.  Then, we cut out both the residue name and number (it's important to get the residue number as well, as you could have two or more residues in a row with the same name.  Then we remove any "WAT or HOH" (water molecules).  Then, we use uniq, which makes a list of just the uniq lines.  Then we count up all the lines.  Simple.</p>

<pre>
grep -v REMARK filename.pdb | grep "ATOM\|HETATM" | cut -b18-26 | grep -v "WAT\|HOH" | uniq | less
grep -v REMARK filename.pdb | grep "ATOM\|HETATM" | cut -b18-26 | grep -v "WAT\|HOH" | uniq | wc -l
</pre>

<p><b>Count the number of each type of atom in a .pdb file</b><br />
This one is a lot simpler, just cut out the atom type field, sort them and use uniq with the -c flag, to count all the occurences.  Neat. <br />
<pre><br />
grep "^ATOM\|^HETATM" 1e3m.pdb | cut -c13-14 | sort | uniq -c<br />
</pre></p>
