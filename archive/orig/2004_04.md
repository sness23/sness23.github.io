<div class="blogbody">
<a name="001161"></a>
<h3 class="title"></h3>

<p>Thank you <a href="http://www.ccs.uky.edu/~mahesh/unixtips.html">Unix Guru List</a>.  You rocked my world by showing me how to do multiple commands in a find command:</p>

<p>Their's:</p>

<pre>
find . -xdev -type d -exec /bin/echo -n {} \; -exec sh -c "ls {} | wc -l" \;
</pre>

<p>But, really, stripped down to it's basics, to run multiple commands, it's:</p>

<pre>
find . -exec sh -c "ls {} | wc -l" \;
</pre>

<p>Sweetness.  Find running multiple commands in a pipe-d exec.  This does a neat trick, it runs "sh" (the Bourne Shell (probably the bash shell on your box)) with the -c argument, meaning, take the next string as the script to execute.</p>

<p>Ok, now I can take over the world.</p>

<h3 class="title">find rocks.</h3>

<p>OK, my killer find command of the day.  Oh yeah, "alias c clear".</p>

<pre>
c ; find . -name '*_arp_warp.log' -cmin -10 -exec sh -c 'grep "Building Cycle" {} | tail -1' \;
</pre>

<p>Cool.</p>

<p>This one looks through a CCP4i directory for running arp_warp jobs and tells you what cycle each one is in.  Add a "-print" or "-printf" (before the -exec) for more fun.</p>
